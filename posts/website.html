On this website

This website is inspired by motherfuckingwebsite.com. See the source here!


Prior to hearing this good word, my posts were "powered by Hugo". Some framework for creating static websites. As someone who has not done much web development in the past, I did not care to learn it. Hugo seemed simple--convert some markdown into HTML and apply some style to it. I don't know how it worked nor how to change things on the site to better fit my liking. This implied that I needed to do one of the following.

  • Learn the framework
  • Make a website using something else
  • Do nothing

For a long while, the third option was my go-to. However, adopting that mindset made me not want to touch the website at all. Clearly a bad approach. So, today, I decided to go minimal and just cook up posts in HTML. No frameworks, no JavaScript, just content. What does this entail?
Honestly, just a few hours of puttering around and trying things out. A single style sheet lives in /style.css to create some very basic formatting. A list of the form 
<ul style="padding-left:0em">
    <li style="display:inline"><a href="/posts.html">POSTS</a></li>
    <li style="display:inline"><a href="/contact.html">CONTACT</a></li>
    <li style="display:inline"><a href="/index.html">ABOUT</a></li>
</ul>
lives at the top of each document for basic navigation. Otherwise, the content is just drafted in HTML in a text editor. I had hesitation about just writing HTML files, but, after a little practice, it is trivial to get the hang of.
